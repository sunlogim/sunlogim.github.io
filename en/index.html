SunLogin Remote Connection / Lag Troubleshooting Lookup

The two real headaches with SunLogin remote control are can't connect and lag. Look up your symptom (e.g. device offline, connection timeout, wrong access password, firewall block, can see but can't control, high latency, free 30fps): first the L1 official checks, then the L2 network-layer root cause (P2P direct vs server relay, NAT traversal, port ranges), then real-world cases. Need the desktop client? Get the SunLogin official download.

Step 1: run the official checks (L1 - do these 4 first)

SunLogin's official connect-failure checklist is just 4 steps, all inside the app: (1) Account & device binding - controller and host log in to the same SunLogin account; (2) Device state & network - the host is powered on, online and not asleep (use “Wake” for a sleeping host, enable it in advance); (3) Firewall & permissions - add SunLogin to the firewall trust list, turn on “Allow remote control” and “Unattended access”, set the access password correctly; (4) Version & restart - update both ends to the latest, restart the client, clear the cache. A fuller walkthrough: SunLogin connection troubleshooting guide.

Step 2: still laggy / still can't connect -> network-layer root cause (L2)

Key idea: can't-connect usually stalls at “establishing the channel”, while lag usually means you went over relay. SunLogin first tries P2P traversal to link the two ends (lowest latency, 20-50ms); only if that fails does it fall back to the vendor's server relay (80-200ms) - the client shows “P2P” or “Relay” in the lower-left. The diagram is a connection self-check panel; deeper causes: symmetric NAT / router UPnP, enterprise DPI and port ranges, and frame rate being capped by the monitor refresh rate.

Real-world cases - quick read

  • The office PC just won't connect but a phone hotspot connects instantly: usually the office network's traffic management / DPI is blocking remote-control traffic - have IT whitelist it, or put the host on a hotspot temporarily.
  • Connected but the mouse ghosts and latency tops 100ms: the lower-left showing “Relay” is the cause - enable UPnP on your router to win P2P; in practice latency can drop from ~120ms to ~35ms.
  • The access password keeps coming back wrong: the access password is NOT your account login password - reset the access password on the host, then connect.
  • Device shows online yet the connection times out: often an expired login token / the host slept and lost network - re-login on the host, or enable “Wake” and unattended access in advance.
  • Window dragging clearly stutters yet latency isn't high: the free tier caps at 30fps - upgrade to a paid tier for 60fps, or the gaming tier for up to 144fps (needs a matching monitor).
  • Black screen, can only see but not operate: usually a missing permission (macOS needs Screen Recording / Accessibility), or the host's CPU / RAM is maxed so encoding can't keep up.
