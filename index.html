向日葵官方给的连接失败排查就 4 步、全程在软件内：① 账号与设备绑定——主控与被控登录同一个向日葵账号；② 设备状态与网络——被控端开机联网、未休眠（休眠可用「唤醒功能」，需提前开启）；③ 防火墙与权限——把向日葵加入防火墙信任、开启「允许远程控制」和「无人值守访问」、访问密码正确；④ 版本与重启——两端都更新到最新版、重启客户端、清缓存。更系统的图文见 向日葵连接失败排查教程。

第二步：连上了还卡 / 还连不上 → 网络层深度真因（L2）

关键认知：连不上多半卡在「建立通道」、卡顿多半卡在「走了转发」。向日葵先用 P2P 穿透 把两端打通（延迟最低 20–50ms），打不通才走贝锐服务器转发（延迟 80–200ms）——客户端左下角会显示「P2P」或「转发」。下图是连接自检面板示意；深层判因还要看：对称 NAT / 路由器 UPnP、企业网 DPI 与端口段、帧率受显示器刷新率限制。

真实场景速判

  • 公司电脑死活连不上、用手机热点却秒连：多半是公司网的上网行为管理 / DPI 拦了远控流量，让 IT 放行白名单，或被控端临时走手机热点。
  • 连上了但鼠标拖影、延迟过百：左下角显示「转发」就是根源——登录路由器开启 UPnP 争取 P2P，实测延迟可从 ~120ms 降到 ~35ms。
  • 访问密码反复提示错误：访问密码不是登录账号的密码，到被控端「设置」里重设访问密码再连。
  • 设备显示在线却连接超时：常是登录 token 过期 / 被控端休眠断网，被控端重新登录、或提前开「唤醒」与无人值守。
  • 拖窗口明显卡但延迟并不高：免费版帧率上限 30 帧，升级付费可到 60 帧、游戏版最高 144 帧（需显示器支持）。
  • 黑屏、只能看不能操作：多为权限没给（macOS 需勾「屏幕录制 / 辅助功能」），或被控端 CPU / 内存跑满导致编码跟不上。
