向日葵遠端連線 / 卡頓故障診斷查詢

向日葵遠端控制最惱人的就是連不上和卡頓。輸入或點選你遇到的症狀（如 裝置離線、連線逾時、存取密碼錯誤、防火牆攔截、看得到無法操作、卡頓延遲高、免費版30幀），先看 L1 官方排查 怎麼做，再看 L2 網路層深度真因（P2P 直連 vs 伺服器轉發、NAT 穿透、連接埠範圍），最後對照真實情境。需要桌面端可到 向日葵遠端控制官網下載。

第一步：依官方口徑排查（L1·先做這 4 步）

向日葵官方給的連線失敗排查就 4 步、全程在軟體內：① 帳號與裝置綁定——主控與被控登入同一個向日葵帳號；② 裝置狀態與網路——被控端開機連網、未睡眠（睡眠可用「喚醒功能」，需提前開啟）；③ 防火牆與權限——把向日葵加入防火牆信任、開啟「允許遠端控制」和「無人值守存取」、存取密碼正確；④ 版本與重啟——兩端都更新到最新版、重啟用戶端、清快取。更系統的圖文見 向日葵連線失敗排查教學。

第二步：連上了還卡 / 還連不上 → 網路層深度真因（L2）

關鍵認知：連不上多半卡在「建立通道」、卡頓多半卡在「走了轉發」。向日葵先用 P2P 穿透 把兩端打通（延遲最低 20–50ms），打不通才走貝銳伺服器轉發（延遲 80–200ms）——用戶端左下角會顯示「P2P」或「轉發」。下圖是連線自檢面板示意；深層判因還要看：對稱 NAT / 路由器 UPnP、企業網 DPI 與連接埠範圍、幀率受螢幕更新率限制。

真實情境速判

  • 公司電腦死活連不上、用手機熱點卻秒連：多半是公司網的上網行為管理 / DPI 攔了遠控流量，讓 IT 放行白名單，或被控端臨時走手機熱點。
  • 連上了但滑鼠拖影、延遲破百：左下角顯示「轉發」就是根源——登入路由器開啟 UPnP 爭取 P2P，實測延遲可從 ~120ms 降到 ~35ms。
  • 存取密碼反覆提示錯誤：存取密碼不是登入帳號的密碼，到被控端「設定」裡重設存取密碼再連。
  • 裝置顯示在線卻連線逾時：常是登入 token 過期 / 被控端睡眠斷網，被控端重新登入、或提前開「喚醒」與無人值守。
  • 拖視窗明顯卡但延遲並不高：免費版幀率上限 30 幀，升級付費可到 60 幀、遊戲版最高 144 幀（需螢幕支援）。
  • 黑畫面、只能看不能操作：多為權限沒給（macOS 需勾「螢幕錄製 / 輔助使用」），或被控端 CPU / 記憶體跑滿導致編碼跟不上。
