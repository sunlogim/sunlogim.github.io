SunLogin 원격 연결 / 끊김 진단 조회

SunLogin 원격 제어에서 가장 골치 아픈 두 가지는 연결 불가와 끊김입니다. 겪고 있는 증상(예: 기기 오프라인, 연결 시간 초과, 접속 비밀번호 오류, 방화벽 차단, 보이지만 제어 불가, 높은 지연, 무료판 30fps)을 입력해 먼저 L1 공식 점검을 보고, 다음으로 L2 네트워크 계층 심층 원인(P2P 직결 vs 서버 중계, NAT 통과, 포트 대역)을 확인한 뒤 실제 사례와 대조하세요. 데스크톱 클라이언트가 필요하면 SunLogin 원격 제어 공식 다운로드를 이용하세요.

1단계: 공식 방향으로 점검(L1·먼저 이 4단계)

SunLogin 공식 연결 실패 점검은 4단계이며 모두 앱 안에서 끝납니다. ① 계정·기기 바인딩—제어 측과 피제어 측이 같은 SunLogin 계정으로 로그인; ② 기기 상태·네트워크—피제어 측이 켜져 있고 인터넷 연결·절전 아님(절전 시 「깨우기」 사용, 미리 활성화 필요); ③ 방화벽·권한—SunLogin을 방화벽 신뢰 목록에 추가하고 「원격 제어 허용」·「무인 접속」을 켜고 접속 비밀번호를 정확히; ④ 버전·재시작—양쪽 모두 최신으로 업데이트, 클라이언트 재시작, 캐시 정리. 더 자세한 안내는 SunLogin 연결 실패 점검 가이드를 보세요.

네트워크 / 통과연결 시간 초과 · 방화벽 차단 · 회사망 · 중계 / P2P
채널 수립 실패 또는 중계로 인한 높은 지연. 먼저 방화벽 신뢰 + 네트워크 교체; 심층으로는 P2P 통과 여부, 기업망 DPI와 NAT 유형을 봅니다.
계정 / 권한기기 오프라인 · 접속 비밀번호 오류 · 계정 불일치 · 깨우기 실패
피제어 측 온라인 + 양쪽 동일 계정이 전제. 접속 비밀번호 ≠ 로그인 비밀번호; 토큰 만료 시 재로그인; 절전은 「깨우기」로.
화면 / 성능검은 화면 · 보이지만 제어 불가 · 끊김 지연 · 흐릿함 · 무료 30fps
연결 후 체감 문제. 권한(화면 녹화 / 손쉬운 사용), 데스크톱 모드, 프레임레이트 등급과 피제어 측 성능이 관건.
버전 / 프로토콜프로토콜 버전 불일치 · 업데이트 후 이상
양쪽 버전 차이가 크면 비호환 쉬움(오류 코드 30); 모두 최신으로, 재시작·캐시 정리.

2단계: 연결돼도 끊기거나 / 여전히 안 되면 → 네트워크 계층 심층 원인(L2)

핵심: 연결 불가는 대개 「채널 수립」에서 멈추고, 끊김은 대개 「중계로 갔다」는 뜻입니다. SunLogin은 먼저 P2P 통과로 양쪽을 직결(최저 지연 20–50ms)하고, 안 되면 벤더 서버 중계(80–200ms)로 넘어갑니다—클라이언트 좌측 하단에 「P2P」 또는 「중계」가 표시됩니다. 아래는 연결 자가 점검 패널 예시이며, 심층 원인으로는 대칭 NAT / 라우터 UPnP, 기업망 DPI와 포트 대역, 모니터 주사율에 묶이는 프레임레이트가 있습니다.

실제 사례 빠른 판단

  • 회사 PC는 죽어도 연결 안 되는데 휴대폰 핫스팟은 바로 연결: 대개 회사망의 인터넷 행위 관리 / DPI가 원격 트래픽을 막은 것. IT에 화이트리스트 허용을 요청하거나 피제어 측을 임시로 휴대폰 핫스팟에 연결하세요.
  • 연결은 됐지만 마우스 잔상, 지연 100ms 초과: 좌측 하단에 「중계」가 보이면 그게 원인—라우터에 로그인해 UPnP를 켜 P2P를 노리면 실측 지연이 ~120ms에서 ~35ms로 떨어집니다.
  • 접속 비밀번호가 계속 틀렸다고 표시: 접속 비밀번호는 로그인 계정의 비밀번호가 아닙니다. 피제어 측 「설정」에서 접속 비밀번호를 재설정한 뒤 연결하세요.
  • 기기는 온라인인데 연결 시간 초과: 보통 로그인 토큰 만료 / 피제어 측 절전·네트워크 끊김. 피제어 측에서 재로그인하거나 미리 「깨우기」와 무인 접속을 켜세요.
  • 창을 끄는데 확연히 끊기지만 지연은 높지 않음: 무료판 프레임레이트 상한이 30fps입니다. 유료는 60fps, 게임판은 최대 144fps(모니터 지원 필요).
  • 검은 화면, 보기만 되고 조작 불가: 대개 권한 미부여(macOS는 「화면 녹화 / 손쉬운 사용」 체크 필요), 또는 피제어 측 CPU / 메모리 포화로 인코딩이 못 따라가는 경우.
